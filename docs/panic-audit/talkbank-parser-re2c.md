# Panic audit: talkbank-parser-re2c

**Status:** Reference
**Last updated:** 2026-06-13 21:07 EDT

See [README](README.md) for the shared policy. This page records the
crate-specific panic surface.

## Surface

One module-level allow plus a small number of inline sites.

### Module-level: the generated lexer

`src/lib.rs` carries a module-scoped
`#![allow(clippy::panic, clippy::unwrap_used, clippy::expect_used)]` on the
`lexer` module. That module is the re2c-generated DFA (~63k lines) included
via `include!(concat!(env!("OUT_DIR"), "/lexer.rs"))`. Its single
`panic!("internal lexer error")` sits at the unreachable-by-construction
terminal state and is intrinsic to re2c's output shape. The file is
regenerated by `build.rs` on every build, so the panic cannot be annotated
per-site; the module-level allow with an explanatory comment is the only
option.

### Inline: parser layer

- **`retrace_kind().expect("is_retrace was true")`** (`parser/classify.rs`,
  `parser/main_tier.rs`): the `expect` runs only inside a branch already
  gated by `ann.is_retrace()`, so `retrace_kind()` is `Some` by
  construction.
- **`panic!("token_to_parsed_annotation: unexpected ...")`**
  (`parser/classify.rs`): a genuine internal-invariant guard. The function
  is only ever called with annotation tokens; any other token kind is an
  internal bug in the caller, and crashing loudly is correct rather than
  fabricating a model value.

cargo clippy -p talkbank-parser-re2c --lib --bins --locked -- -D warnings
