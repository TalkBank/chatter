# Retained validation fixture (not a generated corpus)

**Status:** Reference
**Last updated:** 2026-06-16 20:46 EDT

This directory once held a hand-written validation-error corpus emitted by the
`generate_error_corpus` binary (`crates/talkbank-parser-tests/src/bin/`). That
generator's validation half was retired once `spec/errors/` became the single
source of truth for validation tests: the spec-driven corpus is generated by
`gen_validation_corpus` into
`crates/talkbank-parser-tests/tests/error_corpus/validation_errors/` (a
crate-relative path, distinct from this repo-root directory) and consumed by the
data-driven runner `crates/talkbank-parser-tests/tests/validation_error_corpus.rs`.

`E601_invalid_dependent_tier.cha` is **retained as a committed, hand-maintained
fixture**, not generated by any tool. Two cross-crate integration tests depend
on its exact content and line layout:

- `crates/talkbank-transform/tests/render_parity.rs` (error line-numbering parity)
- `apps/chatter-desktop/src-tauri/tests/validation_bridge.rs` (desktop bridge)

Do not delete or regenerate this file. Edit it only alongside those tests.
