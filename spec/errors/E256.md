# E256: Illegal curly single quote

## Description

A curly single quotation mark (U+2018 or U+2019) is used as a word character.
CHAT requires the ASCII apostrophe; the curly form (typically a typographic
apostrophe introduced by autocorrect or ASR) is not a legal word character.
Mirrors CLAN CHECK errors 138 (U+2019) and 139 (U+2018).

## Metadata

- **Error Code**: E256
- **Category**: parser
- **Level**: word
- **Layer**: parser
- **Status**: implemented

## Example 1

**Trigger**: A word containing a curly single quote (U+2019)
**Expected Error Codes**: E256

```chat
@UTF8
@Begin
@Languages:	eng
@Participants:	CHI Target_Child
@ID:	eng|corpus|CHI|||||Target_Child|||
*CHI:	don’t .
@End
```

## Expected Behavior

The grammar must not treat the curly single quote as a legal word character.
The parser should recognize it as an illegal-character node and report E256
rather than silently absorbing it into the word token. The curly double quotes
(U+201C/U+201D) remain legal only as quotation-group bookends, never as word
characters.

## CHAT Rule

CHAT word text uses ASCII punctuation; curly typographic single quotes are not
legal word characters and should be the ASCII apostrophe. CLAN CHECK 138/139.
CHAT manual: https://talkbank.org/0info/manuals/CHAT.html

## Notes

- Both parsers reject the curly single quote. The tree-sitter grammar recognizes
  it as a dedicated `illegal_curly_quote` node (excluded from `word_segment` via
  the symbol registry) and the parser emits E256 with a precise span; the re2c
  lexer emits a recognized `IllegalCurlyQuote` token that the file-level parser
  reports as E256. In both, the offending quote is dropped and the surrounding
  words survive, so validation continues.
